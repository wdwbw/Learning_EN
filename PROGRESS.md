- 每代理讀 `SEED_WORDS.slice(START,START+30)`,翻成粵語,寫 `trans/y_<START>.json`。
- **已完成:索引 0–90(y_0000/y_0030/y_0060)。待做:索引 90–3090。**

### 2. 英文擴到 5000 → 10000(未開始)
- 管線:新主題批次寫 `words_raw/*.json` → `node tools/assemble-all.js`(全域去重、重編 id、產 `words-pack1.js`)。
- **新增的英文字要再補全部翻譯**(德/法/日/粵),否則非英語模式看不到那些字。
- 目前 3096 → 5000 約 +1900 字;→10000 約 +6900 字。

### 3. 新字翻譯的注意
- `assemble-all.js` 會**依字母重新編 id**(p0001…),所以擴充英文後,舊翻譯的 id 對應「大多」不變(因為多為既有字),但若新字插入改變排序,少數 id 可能位移 → 建議**擴英文後,以「英文字面 word」為準重新核對翻譯覆蓋**,或改用穩定 id 策略(見下「已知風險」)。

## Token 估算(本專案實測)
- 英文生成 ≈ 900 tokens/字;三語翻譯 ≈ 1,600 tokens/字(每多一個語言約 +530/字)。
- 廣東話補齊 3096 字 ≈ 3096 × ~550 ≈ **~1.7M**。
- 英文 →5000(+1900)四語(含粵五語)完整 ≈ **~6–7M**。
- **帳號用量上限**:大批 fan-out 一輪常跑不完(訊息會顯示重置時間),分批 / 跨額度週期續跑;所有組裝腳本冪等,重跑不重複。

## 已知風險 / TODO 改善

- **圖片**:用 loremflickr 動態抓,離線/無網時不顯示(已 graceful 隱藏)。
- **口說**:粵語/日語逐字比對、德法保留重音;需 HTTPS + 手機 Chrome 最穩。
- **Service Worker**:改 `index.html`/`words*.js`/`langs-pack1.js` 後,若手機開過要清快取或升 `sw.js` 的 `CACHE` 版號。
